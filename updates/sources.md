# Canonical sources to monitor

The URLs the update radar fetches on each refresh. Keep this list curated.

## dlt (dlthub)
- Release notes / changelog: https://dlthub.com/docs (see "Reference → Changelog") and
  GitHub releases: https://github.com/dlt-hub/dlt/releases
- Databricks destination docs: https://dlthub.com/docs/dlt-ecosystem/destinations/databricks
- Blog: https://dlthub.com/blog

## dbt + dbt-databricks
- dbt platform release notes: https://docs.getdbt.com/docs/dbt-versions/dbt-cloud-release-notes
- Annual release notes (e.g. 2025/2026): https://docs.getdbt.com/docs/dbt-versions/2025-release-notes
- dbt-databricks adapter releases: https://github.com/databricks/dbt-databricks/releases
- About Fusion: https://docs.getdbt.com/docs/fusion/about-fusion
- "What's shipped in dbt" blog: https://www.getdbt.com/blog

## Databricks (Lakeflow / Unity Catalog)
- Lakeflow Spark Declarative Pipelines (ex-DLT) release notes:
  https://docs.databricks.com/aws/en/release-notes/dlt/2026
- "What happened to Delta Live Tables (DLT)?":
  https://docs.databricks.com/aws/en/ldp/where-is-dlt
- Platform release notes: https://docs.databricks.com/aws/en/release-notes/
- Databricks blog: https://www.databricks.com/blog

> Swap `aws` for `azure`/`gcp` in docs.databricks.com paths to match your cloud.
